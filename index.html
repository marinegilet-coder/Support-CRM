📖 Process de traitement d'une demande

Workflow standard à appliquer dès réception d'une demande utilisateur. Temps cible : 3 à 5 min de saisie par ticket.

1
🟢 Réception de la demande (≈ 30 sec)
La demande arrive par mail. Chloé clique immédiatement sur "+ Nouveau ticket".
Canal d'entrée :
  • 📧 Mail sur Chloe.morio@artemiscourtage.com
5 champs à renseigner : Courtier (auto-complétion) → Type → Module → Titre court → Description (copier-coller du mail OK).
Région / Vague / Agence / Date sont auto-remplis depuis le profil courtier.
2
🟡 Qualification (≈ 1 min)
Chloé évalue la priorité et l'action à mener.
Grille de priorité :
  • 🔴 Bloquant : empêche un courtier de travailler (impossible de créer un dossier, SSO Bewai KO, perte de données)
  • 🟠 Gênant : workaround possible mais perte de temps significative
  • 🟢 Mineur : question d'usage, suggestion, point cosmétique
Décision d'orientation :
  • ✅ Question simple → réponse directe (FAQ existante)
  • 🔧 Bug confirmé → escalade JIRA Euodia
  • 💡 Évolution → backlog produit (à voir avec Marine)
  • 📚 Question récurrente (3ème occurrence) → à ajouter en FAQ Lemon Learning
3
🟠 Traitement
Selon l'action retenue à l'étape 2.
  • Réponse directe : mail au courtier (réponse-type adaptée), ticket → "Résolu"
  • Escalade JIRA : créer le ticket dev avec lien vers le ticket support, copier le n° JIRA, statut → "En attente dev"
  • FAQ : flag "À capitaliser", traité en batch le vendredi
  • Évolution : flag "Backlog produit", point bi-mensuel avec Marine
SLA cibles :
  • 🔴 Bloquant : réponse < 2h, résolution < 24h
  • 🟠 Gênant : réponse < 4h, résolution < 5j
  • 🟢 Mineur : réponse < 24h, résolution < 15j
4
🔴 Clôture (≈ 30 sec)
Renseigner la solution et fermer le ticket pour alimenter le tableau de bord.
  • Date de clôture (auto si statut → "Résolu")
  • Solution apportée : 1-2 lignes (utile pour identifier les patterns récurrents)
  • Notification automatique au courtier
  • Si bug résolu par dev : confirmer avec le courtier que ça fonctionne avant clôture

🪶 Optimisations pour alléger la saisie

A
Templates de tickets récurrents
Pour les questions ultra-fréquentes (ex : "Comment réinitialiser mon mot de passe", "Pourquoi je ne vois pas mon dossier"), créer des tickets pré-remplis en 1 clic.
B
Bibliothèque de réponses-types
20 réponses copiables qui couvrent ~60% des cas. Personnalisation rapide (nom courtier, contexte) puis envoi.
C
Capitalisation hebdomadaire
Tous les vendredis : passer en revue les questions de la semaine, identifier les 3 plus fréquentes, créer/enrichir les articles Lemon Learning correspondants. Objectif : réduire le volume au fil du temps.
📁 Gestion des données

Les données sont sauvegardées automatiquement dans le navigateur (localStorage). Tu peux exporter le fichier JSON pour le sauvegarder ailleurs, l'envoyer à un collègue, ou l'importer pour récupérer une version antérieure.

💡 Conseil : exporter le JSON chaque fin de semaine et le stocker sur Google Drive partagé avec Marine, en cas de souci navigateur.
